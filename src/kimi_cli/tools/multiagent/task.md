生成一个子代理来执行特定任务。子代理将在一个全新的上下文中生成，不包含你任何历史记录。

**上下文隔离**

上下文隔离是使用子代理的主要好处之一。通过将任务委托给子代理，你可以保持主上下文的清晰，并专注于用户请求的主要目标。

以下是一些你可能希望使用此工具进行上下文隔离的场景：

- 你编写了一些代码，但它没有按预期工作。在这种情况下，你可以生成一个子代理来修复代码，并要求子代理返回修复方式。这可能很有益，因为修复代码的详细过程可能与你的主要目标无关，并可能使你的上下文混乱。
- 当你需要特定库、框架或技术的最新知识来继续你的任务时，你可以生成一个子代理在互联网上搜索所需信息，并向你返回收集到的相关信息，例如代码示例、API 参考等。这可以避免在你的上下文中出现大量不相关的搜索结果。

不要直接将用户提示转发给 Task 工具。不要简单地为每个待办事项生成 Task 工具。这会让用户感到困惑，因为用户无法看到子代理在做什么。只有你可以看到子代理的响应。因此，只为非常具体和狭窄的任务生成子代理，例如修复编译错误或搜索特定解决方案。

**并行多任务处理**

并行多任务处理是此工具的另一个主要好处。当用户请求涉及多个相互独立的子任务时，你可以在单个响应中多次使用 Task 工具，让子代理为你并行工作。

示例：

- 用户请求在项目中编写、重构或修复多个模块/文件，并且它们可以独立测试。在这种情况下，你可以生成多个子代理，每个子代理处理不同的模块/文件。
- 当你需要分析一个庞大的代码库（> 数十万行）时，你可以生成多个子代理，每个子代理探索代码库的不同部分并收集总结结果。
- 当你需要搜索多个网页查询时，你可以生成多个子代理以提高效率。
